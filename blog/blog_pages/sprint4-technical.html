How confident you feel using problem solving techniques and process?

  • Trying something:

    • This was always how I will start my problem solving process. I will look over the code to see if I understand it. If I do, I will start working through the code with what I feel needs to be done to complete the code. If unsuccessful I start going through other problem solving techniques.

  • Console.logging

    • I found console logging a VERY effective way to understand what is happening and how to break things down, especially in the Kata exercises.
    If I could find a way to create a variable so that I could console log what I was going through it would make solving the problem far easier than not being able to do that.

  • Reading error messages

    • This is absolutely necessary to understand what is going wrong with you code and where. This can tell you what errors are happening in your code to make it incomplete and on which lines the console is having problem with.

  • Googling

    • Always the best option for problem solving if you don't have anyone to talk to and have an internet connection. There's a high chance that someone has run into the same or a similar problem to what you have online. Searching what you need to through google or looking at YouTube videos to learn is a very effective problem solving technique and one that I know many (even experienced) coders do regularly.

  • Asking coaches for help

    • When available it's always a great option to talk to someone directly about the problems you are having who have plenty of experience coding. Unlike Google where you really need to define what your problem is or may not know exactly what the problem is, talking to someone directly and showing the code can be (if not is) the most effective way of getting through a problem.

  • Improving your process with reflection

    • This is the strongest way to learn and understand problems once they have been solving. If you can understand and work through the problem then you will be able to solve similar problems (or at least work through them more effectively than not) than if you just copy and pasted and/or finished the problem straight after that.
